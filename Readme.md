# Weather ML Pipeline

Проект представляет собой автоматизированный ML-пайплайн для прогнозирования температуры на основе данных погоды с использованием Apache Airflow. Система ежедневно собирает данные с Open-Meteo API, обучает модель случайного леса и выполняет инференс с визуализацией.

## Компоненты

- Apache Airflow – оркестрация задач.
- Open-Meteo API – источник данных о погоде.
- Scikit-learn – обучение модели RandomForestRegressor.
- Pandas / NumPy – обработка данных.
- Matplotlib – визуализация результатов.
- Docker / Docker Compose – контейнеризация.

## Структура проекта


```

lab3/
├── dags/                   # DAG-файлы Airflow
├── weather_pipeline/       # Логика ETL и обучения
├── data/                   # Данные, модели, метрики, графики
├── logs/                   # Логи
├── docker-compose.yaml     # Конфигурация контейнеров
├── .env                    # Переменные окружения
└── requirements.txt        # Зависимости

```

## Работа пайплайна

1. DAG weather_ml_pipeline_enhanced (ежедневно):
   - Загрузка данных через fetch_weather.
   - Препроцессинг (лаги, признаки, разбиение на train/test).
   - Обучение модели и сохранение артефактов.
   - Проверка качества (R2 >= 0.7).
   - Триггер инференса.

2. DAG weather_inference_demo:
   - Проверка качества данных.
   - Инференс и сохранение графика предсказаний.

## Инструкция по запуску

Предварительно должны быть установлены Docker и Docker Compose.

1. Клонируйте репозиторий:
   ```bash
   git clone <url>
   cd lab3

```

2. Проверьте настройки в .env:
```env
AIRFLOW_UID=50000
_AIRFLOW_WWW_USER_USERNAME=admin
_AIRFLOW_WWW_USER_PASSWORD=admin

```


3. Запустите проект:
```bash
docker-compose up -d

```



Веб-интерфейс доступен по адресу http://localhost:8080. Логин и пароль: admin / admin.

## Результаты

Метрики модели (MAE, RMSE, R2) сохраняются в директорию data/metrics/ в формате JSON. Графики предсказаний сохраняются в data/plots/.
